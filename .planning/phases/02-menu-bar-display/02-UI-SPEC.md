# Phase 2 — UI Design Contract

> Visual and interaction contract for the macOS menu bar display. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Platform note:** This phase is a native macOS menu bar application using AppKit (NSStatusItem + NSMenu). There are no web technologies, no component registries, and no CSS. All visual specifications reference AppKit APIs and macOS system conventions.

| Property | Value |

## Spacing Scale

The menu bar is a constrained rendering surface. macOS controls the vertical height (22pt on standard displays). Horizontal spacing is managed within the status item text string itself.

| Token | Value | Usage |
|-------|-------|-------|
| arrow-gap | 1 space character (U+0020) | Space between arrow glyph and numeric value: `↑ 1.2` |
| value-unit-gap | 1 space character (U+0020) | Space between numeric value and unit: `1.2 MB/s` |
| direction-separator | 1 space character (U+0020) | Space between upload and download in "Both" mode: `↑ 1.2 MB/s ↓ 456 KB/s` |
| menu-item-indent | 0px | NSMenu items use system-default indentation, no custom override |
| menu-separator | NSMenuItem.separator() | System-standard separator line between menu item groups |

Exceptions: This phase has no pixel-based custom spacing. All spacing is character-level within a text string or system-controlled by NSMenu/NSStatusBar. The 8-point grid does not apply to menu bar text rendering.

## Typography

Phase 2 renders text in exactly two contexts: the status bar button and the context menu. Both use system fonts controlled by AppKit.

| Role | Font API | Size | Weight | Line Height |
|------|----------|------|--------|-------------|
| Status bar speed text | `NSFont.monospacedDigitSystemFont(ofSize:weight:)` | `NSFont.systemFontSize` (~13pt) | `.regular` (400) | System-controlled (single line) |
| Menu item text | System default (no override) | System default (~13pt) | System default (`.regular`) | System-controlled |
| Menu item disabled text | System default | System default | System default | System-controlled |

**Why monospacedDigitSystemFont:** Digits 0-9 render at equal width (tabular figures), preventing horizontal jitter when speed values change (e.g., "9.9" to "10.0"). Letters and symbols remain proportionally spaced. This directly satisfies BAR-04.

**Font size rationale:** `NSFont.systemFontSize` matches other menu bar items (clock, battery %). If it appears oversized after implementation, fall back to `NSFont.systemFontSize - 1` (approximately 12pt). Do not go below 12pt — readability degrades on non-Retina displays.

## Color

Phase 2 uses **zero custom colors**. The menu bar inherits all colors from macOS system appearance.

| Role | Value | Usage |
|------|-------|-------|
| Status bar text | `NSColor.controlTextColor` (inherited) | Speed text in menu bar — automatically adapts to light/dark mode and wallpaper tinting |
| Menu item text | System default | NSMenu items inherit system text color |
| Menu item disabled text | System default (grayed) | "Metrics" and "Preferences" disabled items |
| Menu background | System default | NSMenu uses system-standard vibrancy and blur |
| Accent | System accent color (user-configured) | Menu item highlight on hover — system-controlled, no override |

**Why no custom colors:** The macOS menu bar is a system-controlled surface. Custom colors would violate platform conventions, look out of place next to system menu bar items (Wi-Fi, Battery, Clock), and break with wallpaper tinting on macOS Sonoma+. The standard About panel also uses system colors exclusively.

Accent reserved for: Not applicable — macOS controls all accent usage in NSMenu highlight states.

## Copywriting Contract

### Status Bar Text Formats

All text formats are locked by CONTEXT.md decisions. The executor MUST implement these exactly.

| State | Display Text | Decision Ref |
|-------|-------------|--------------|
| Auto mode, download dominant | `↓ 1.2 MB/s` | D-06, D-01 |
| Auto mode, upload dominant | `↑ 1.2 MB/s` | D-06, D-01 |
| Auto mode, equal or both zero | `↑ 0 KB/s` | D-06 (upload wins tie) |
| Upload only mode | `↑ 1.2 MB/s` | D-07 |
| Download only mode | `↓ 1.2 MB/s` | D-07 |
| Both mode | `↑ 1.2 MB/s ↓ 456 KB/s` | D-08 |
| Zero/near-zero traffic (< 1 KB/s) | `↑ 0 KB/s` or `↓ 0 KB/s` | D-10 |
| No active interfaces | `—` (em dash, U+2014) | D-18 |

### Numeric Formatting Rules

| Rule | Specification | Decision Ref |
|------|---------------|--------------|
| Precision < 100 | 1 decimal place: `1.2`, `45.3`, `99.9` | D-02 |
| Precision >= 100 | 0 decimal places: `100`, `456`, `1024` | D-02 |
| Unit labels | `KB/s`, `MB/s`, `GB/s` (base-10 SI byte-rate) | D-03 |
| Auto-scale boundary KB | >= 1,000 B/s | D-09 |
| Auto-scale boundary MB | >= 1,000,000 B/s | D-09 |
| Auto-scale boundary GB | >= 1,000,000,000 B/s | D-09 |
| Below 1 KB/s | Always show `0 KB/s` — no B/s tier | D-10 |
| Fixed unit ceiling | Selected unit is max; smaller values fall back | D-11 |

|-----------|--------|------|
| No active network interfaces | Status bar text | `—` (em dash) — communicates "nothing to show" without alarming the user |

### Error States

| Condition | Visual | Behavior |
|-----------|--------|----------|

| NetworkMonitor fails to start | Status bar shows `—` | Em dash persists; logged via `os.Logger` |
| sysctl read error on a specific interface | Speed shows as zero for that interface | Handled in Phase 1 SpeedComputation; Phase 2 just displays the result |

## Text Width Stability (BAR-04)

This section documents the anti-jitter contract required by BAR-04.

| Technique | Purpose | Scope |
|-----------|---------|-------|
| `NSFont.monospacedDigitSystemFont` | All digits (0-9) render at identical width | Applied via `NSAttributedString` on every title update |
| Auto display mode (D-06) | Shows only one direction at a time | Keeps string shorter; fewer characters to shift |
| Adaptive precision (D-02) | Value transitions (9.9 to 10.0) change digit count by 1 | Accepted tradeoff — monospacedDigit keeps the shift minimal |
| Unit transitions (KB/s to MB/s) | Unit label width changes slightly | Accepted tradeoff — happens infrequently (at clean SI boundaries) |

**Accepted width instability:** Minor width changes at unit boundaries (KB/s vs MB/s) and digit-count transitions (99.9 to 100) are inherent to variable-length numeric display and are standard practice in macOS network monitors. The monospacedDigit font eliminates the more disruptive per-update jitter from proportional digit widths.
