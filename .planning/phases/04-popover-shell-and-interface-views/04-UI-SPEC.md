# Phase 4 — UI Design Contract

> Visual and interaction contract for the popover shell and interface views. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (native macOS) |
| Preset | not applicable |
| Component library | SwiftUI + AppKit (NSPopover, NSStatusItem) |
| Icon library | SF Symbols (Apple system icons) |
| Font | SF Pro (macOS system font via SwiftUI `.body`, `.headline`, etc.) |

**Rationale:** This is a native macOS menu bar app. The design system is Apple's Human Interface Guidelines. All styling uses SwiftUI semantic types and system-provided values. No custom design tokens or third-party UI libraries.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | SwiftUI Equivalent | Usage |
|-------|-------|--------------------|-------|
| xs | 4px | `.padding(4)` | Inline icon-to-label gaps, minimal separators, gap between rows |
| sm | 8px | `.padding(8)` | Compact element spacing, inner row padding |
| lg | 16px | `.padding(16)` | Default content padding within sections, popover content insets (top, bottom, leading, trailing), segmented control top margin |
| xl | 24px | `.padding(24)` | Space between aggregate header and interface list |
| 2xl | 32px | `.padding(32)` | Not used in this phase |

Exceptions:
- Row height minimum: 44pt per interface row (accessibility touch target, even though macOS does not require it, maintains comfortable click area)
- Aggregate header bottom padding: 24px to visually separate from interface list

**Source:** Claude's discretion (CONTEXT.md grants discretion over "exact spacing, padding, and font sizes within the popover"). Scale chosen to match macOS HIG density expectations for a utility popover -- tighter than iOS but not cramped. All values are members of the standard spacing set {4, 8, 16, 24, 32, 48, 64}.

## Typography

All typography uses the macOS system font (SF Pro) via SwiftUI text styles. Sizes are specified as SwiftUI `Font` values, which resolve to specific point sizes on macOS.

| Role | SwiftUI Font | Resolved Size (macOS) | Weight | Line Height |
|------|-------------|----------------------|--------|-------------|
| Aggregate speed value | `.title2` | 17pt | `.semibold` (600) | 1.2 |
| Aggregate label | `.subheadline` | 11pt | `.regular` (400) | 1.5 |
| Interface name | `.body` | 13pt | `.regular` (400) | 1.5 |
| Interface speed value | `.body` | 13pt | `.regular` (400) | 1.5 |
| Speed unit suffix | `.body` | 13pt | `.regular` (400) | 1.5 |
| Tab label (segmented) | `.body` | 13pt | `.regular` (400) | system default |
| Placeholder text | `.body` | 13pt | `.regular` (400) | 1.5 |
| Empty state message | `.subheadline` | 11pt | `.regular` (400) | 1.5 |

**Weights used:** 2 weights -- `.regular` (400) for body text, labels, interface names, and speed values; `.semibold` (600) for aggregate speed emphasis only. Interface names achieve visual distinction through their leading position in the row and `.primary` color, not through a separate weight.

**Monospaced digits:** All speed values (aggregate and per-interface) use `.monospacedDigit()` modifier to prevent layout jitter when numbers change. This matches the menu bar pattern established in Phase 2 (StatusBarController uses `NSFont.monospacedDigitSystemFont`).

**Source:** Claude's discretion. Sizes follow SwiftUI text style defaults on macOS, which align with Apple HIG for utility windows.

## Color

This phase uses exclusively semantic SwiftUI colors per D-14. No custom hex values. Automatic dark/light mode adaptation is achieved by using only system-provided color tokens.

| Role | SwiftUI Value | Usage |
|------|---------------|-------|
| Dominant (60%) | `.clear` (window background managed by NSPopover) | Popover background -- NSPopover provides the standard vibrancy/translucency automatically |
| Secondary (30%) | `Color(.controlBackgroundColor)` | Interface row backgrounds (alternating or grouped style) |
| Tertiary | `Color.secondary` | Upload/download labels ("Upload"/"Download"), unit suffixes, interface BSD names |
| Primary text | `Color.primary` | Speed values, interface display names, aggregate totals |
| Accent (10%) | `Color.accentColor` | Reserved for: (1) selected segmented control tab, (2) upload arrow symbol in aggregate header, (3) download arrow symbol in aggregate header |
| Separator | `Color(.separatorColor)` | Divider between aggregate header and interface list |

**Accent reserved for:** Selected tab indicator in segmented control, directional arrow symbols (upload/download) in the aggregate header. Nothing else. Interface rows do not use accent color.

**Destructive:** Not applicable in this phase. Quit action is in the right-click context menu (standard NSMenu), not in the popover. No destructive actions exist within the popover UI.

**Source:** D-14 (locked decision: "Purely semantic SwiftUI colors -- zero custom colors, automatic dark/light mode adaptation").

## Copywriting Contract

| Element | Copy |
|---------|------|
| Aggregate header upload label | (no text label -- use SF Symbol `arrow.up` at 10pt next to value) |
| Aggregate header download label | (no text label -- use SF Symbol `arrow.down` at 10pt next to value) |
| Per-interface upload indicator | (up arrow Unicode `\u{2191}` inline with value, matching menu bar pattern) |
| Per-interface download indicator | (down arrow Unicode `\u{2193}` inline with value, matching menu bar pattern) |
| Metrics tab label | "Metrics" |
| Preferences tab label | "Preferences" |
| Preferences placeholder heading | "Preferences" |
| Preferences placeholder body | "Coming in a future update." |
| Empty state (no interfaces) | "\u{2014}" (em dash, consistent with Phase 2 D-18 pattern) |
| Empty state expanded (popover body) | "No active network interfaces detected." |
| Right-click menu: Metrics | "Metrics" |
| Right-click menu: Preferences | "Preferences" |
| Right-click menu: About | "About Bandwidth Monitor" |
| Right-click menu: Quit | "Quit Bandwidth Monitor" |

**Primary CTA:** None. This phase has no call-to-action buttons. The popover is an informational display opened by clicking the status item. The right-click menu provides navigation ("Metrics", "Preferences") and lifecycle ("Quit Bandwidth Monitor") actions.

**Error state:** Not applicable for this phase. Network monitoring failures are handled silently at the engine layer (Phase 1). The popover displays whatever data `NetworkMonitor.interfaceSpeeds` provides. If monitoring fails, the interface list is empty, which shows the empty state message above.

**Destructive confirmation:** Not applicable. Quit is a standard NSMenu item with keyboard shortcut (Cmd+Q) -- no confirmation dialog required, following macOS convention for menu bar utilities.

**Source:** D-05 (preferences placeholder), D-02 (right-click menu items), CONTEXT.md Claude's discretion (empty state consistent with Phase 2 D-18 em dash approach), D-12 (no quit button inside popover).

## Layout Contract

### Popover Window

| Property | Value | Source |
|----------|-------|--------|
| Width | 400pt | POP-01, D-15 |
| Height | 500pt (content-adaptive, max 500pt) | POP-01, D-15 |
| Popover behavior | `.transient` | Claude's discretion -- closes when user clicks outside, standard for menu bar popovers |
| Popover animation | default NSPopover animation | Claude's discretion |
| Background | Standard NSPopover material (system-managed vibrancy) | D-14 |

### Segmented Control (Tab Switcher)

| Property | Value |
|----------|-------|
| Style | SwiftUI `Picker` with `.pickerStyle(.segmented)` |
| Segments | "Metrics", "Preferences" |
| Default selection | Metrics (D-06) |
| Position | Top of popover, full width with 16px horizontal insets |
| Top margin | 16px from popover top edge |
| Bottom margin | 16px below segmented control before content begins |

### Metrics Tab Layout (top to bottom)

1. **Aggregate Header Section**
   - Full width, 16px horizontal padding
   - Two columns: Upload total (left), Download total (right)
   - Each column: SF Symbol arrow (10pt, accent color) + speed value (`.title2`, `.semibold`, `.monospacedDigit()`)
   - 24px bottom margin before divider

2. **Divider**
   - `Divider()` -- system separator, full width

3. **Interface List**
   - `ScrollView` if content exceeds available height (Claude's discretion)
   - Each row: 44pt minimum height, 16px horizontal padding, 8px vertical padding
   - Row layout: SF Symbol icon (16pt) + 8px gap + display name (`.body`, `.regular`) | upload speed + download speed (right-aligned, `.body`, `.regular`, `.monospacedDigit()`)
   - 4px gap between rows (or use `List` with `.insetGrouped` for natural spacing)
   - Upload/download values stacked vertically per interface or shown inline (inline preferred: `arrow.up value arrow.down value`)

### Interface Row Detail

| Element | Position | Font | Color |
|---------|----------|------|-------|
| SF Symbol (interface type) | Leading, vertically centered | 16pt symbol | `.secondary` |
| Interface display name | After icon, 8px gap | `.body` weight `.regular` | `.primary` |
| Upload speed | Trailing group, top or inline | `.body` `.monospacedDigit()` | `.primary` |
| Download speed | Trailing group, bottom or inline | `.body` `.monospacedDigit()` | `.primary` |
| Arrow indicators | Inline with speed values | `.caption` | `.secondary` |

### SF Symbol Mapping (per D-08)

|----------------|---------------|
| Wi-Fi | `wifi` |

## Interaction Contract

### Click Behavior (D-01)

| Input | Action |
|-------|--------|
| Left-click on status item | Opens NSPopover anchored to status item button, showing Metrics tab (D-06) |
| Left-click while popover open | Closes popover |
| Right-click on status item | Opens NSMenu context menu (D-02) |
| Click "Metrics" in context menu | Opens popover, switches to Metrics tab (D-03) |
| Click "Preferences" in context menu | Opens popover, switches to Preferences tab (D-03) |
| Click outside popover | Closes popover (`.transient` behavior) |
| Esc key while popover focused | Closes popover |

### Data Flow

| Data Source | Consumer | Mechanism |
|-------------|----------|-----------|
| `NetworkMonitor.interfaceSpeeds` | Interface list rows | SwiftUI `@Observable` direct observation |
| `NetworkMonitor.aggregateSpeed` | Aggregate header | SwiftUI `@Observable` direct observation |
| `InterfaceInfo.type` | SF Symbol selection | Switch on `NWInterface.InterfaceType` |
| `InterfaceInfo.displayName` | Row label | Direct string binding |
| `SpeedFormatter` | Speed value formatting | Reuse existing formatter from Phase 2 |

### State Management

| State | Storage | Scope |
|-------|---------|-------|
| Selected tab (Metrics/Preferences) | `@State` in popover root view | View-local by default |
| Tab override from context menu | Shared mutable state (e.g., property on controller or `@Observable` model) | Controller + View |
| Popover shown/hidden | `NSPopover.isShown` | StatusBarController |
