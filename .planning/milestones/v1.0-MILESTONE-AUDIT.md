### Integration Defect

**BandwidthRecorder pollingInterval not synced to user preference**

When the user changes Update Interval in Preferences (1s or 5s), `NetworkMonitor.pollingInterval` is correctly updated. However, `BandwidthRecorder` retains its hardcoded `pollingInterval: 2.0`. Since byte accumulation is `speed * pollingInterval`, recorded values are wrong:
- At 1s polling: bytes doubled (2x inflation)
- At 5s polling: bytes halved (2.5x deflation)
- Error cascades through all aggregation tiers to charts and stats

**Fix:** Sync `BandwidthRecorder.pollingInterval` in the same `UserDefaults.didChangeNotification` observer in AppDelegate, or derive elapsed time from snapshot timestamps instead of assuming a fixed interval.

**Affected:** MON-03, MON-04, POP-02, POP-04

### Architectural Note (Non-Blocking)

`HistoryView.loadChartData()` and `loadStats()` use synchronous `dbWriter.read { }` on the main thread. Acceptable for current data volumes but should use `async` overload for correctness.
