<objective>
Fix the integration defect where BandwidthRecorder retains a hardcoded pollingInterval of 2.0 seconds regardless of the user's chosen update interval preference. When users change the polling interval to 1s or 5s, recorded byte values are currently inflated (2x at 1s) or deflated (2.5x at 5s) because BandwidthRecorder still multiplies speed by 2.0.

Purpose: Ensure data accuracy across all user-configurable polling intervals (MON-03, MON-04), which cascades to correct charts (POP-02) and statistics (POP-04).
Output: Corrected BandwidthRecorder with mutable pollingInterval, AppDelegate wiring, and tests at all three intervals.
</objective>
